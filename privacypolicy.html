8. 📋 Android Permissions Explained

WiFi Server Pro requests minimal, necessary permissions for local file sharing functionality:

8.1 Storage Permissions

📁 File Access (Android 12 and below):
• READ_EXTERNAL_STORAGE
• Used for accessing files to share locally
• Does not transmit data externally
🎵 Media Access (Android 13+):
• READ_MEDIA_IMAGES, READ_MEDIA_VIDEO, READ_MEDIA_AUDIO
• Granular access to specific media types
• Google Play compliant permissions
🗂️ All Files Access (MANAGE_EXTERNAL_STORAGE):
• Google Play Approved Use Case: File Manager/Server functionality
• Core App Function: Essential for comprehensive file sharing server
• User Benefit: Access entire device storage for complete file server capability
• No Alternative: Scoped storage insufficient for full file server functionality
• Transparent Usage: Only used to serve files locally, never transmitted externally
🎯 Selective Access:
• READ_MEDIA_VISUAL_USER_SELECTED
• Android 14+ photo picker integration
• User controls exactly which files to share

8.2 Network Permissions

  • 🌐 INTERNET: Required to create local network servers (does not connect to internet)
  • 📡 ACCESS_NETWORK_STATE: Detect network connectivity for server setup
  • 📶 ACCESS_WIFI_STATE: Get local WiFi network information for IP discovery

8.3 Service Permissions

  • ⚡ FOREGROUND_SERVICE: Keep HTTP/FTP servers running when app is backgrounded
  • 📊 FOREGROUND_SERVICE_DATA_SYNC: Specific service type for file synchronization
  • 🔗 FOREGROUND_SERVICE_CONNECTED_DEVICE: For device-to-device file sharing
  • 🔔 POST_NOTIFICATIONS: Show server status notifications
  • 📳 VIBRATE: Notification vibration feedback
